# 暖心 Todo（Warm Todo）

> 一款温暖治愈风格的移动端待办与习惯管理应用——从设计稿到可运行的单 HTML 原型，完整还原 **21 个交互界面**，并内置 AI 智能助手。

## 🌐 在线预览

**https://besthqhd.github.io/warm-todo-app/** （GitHub Pages 自动部署，每次 push 到 `main` 即更新）

## ✨ 核心特点

- **待办 + 习惯双轮驱动**：任务管理与习惯养成融合，形成完整的自我管理闭环
- **四象限优先级**：内建艾森豪威尔矩阵（重要紧急 / 重要不紧急 / 不重要紧急 / 不重要不紧急），支持多级子任务（无限层级）
- **待办-习惯联动**：完成待办后引导习惯打卡，完成习惯后引导处理待办，双模块自然串联
- **温暖治愈风**：莫兰迪青绿主色 `#5BA898`、米白背景、大圆角、柔和阴影
- **AI 智能助手**：自然语言建待办、智能习惯推荐、自然语言查询、情感鼓励，以及交互式任务拆解（先推理提议再询问确认）
- **长上下文记忆 + RAG**：用户画像与聊天历史持久化注入 LLM；客户端语义检索（embedding）为主，无接口时自动降级为本地词法检索

## 🖥️ 如何预览

**零依赖，浏览器直接打开即可：**

```bash
# 方式一：直接在文件管理器中双击打开
index.html

# 方式二：用 Python 启动本地服务器（推荐，AI 功能需同源服务）
cd warm-todo-app
python -m http.server 8080
# 然后在浏览器访问 http://localhost:8080
```

> 设计基准尺寸 360×798 px（手机框架），页面会根据视口自动等比缩放适配。

## 🤖 AI 后端（可选）

`server/` 提供了一个轻量 Fastify 服务端，为前端 AI 功能提供 Agent 能力：

- **ReAct 工具循环**：模型自主决定先查数据还是先拆解任务，前端可视化工具调用过程
- **RAG 检索**：`rag_search` 工具供 Agent 主动检索历史上下文；索引按语料哈希缓存
- **优雅降级**：无服务端时，前端自动直连 LLM 或降级为本地规则，界面始终可用
- **设置可配**：设置页可配置 Embedding 模型 / 地址，并支持重建索引

```bash
cd server
npm install
npm start        # 默认 http://localhost:3000
```

前端通过 `callAgent` 桥接后端；未启动后端时自动降级，不影响基础交互。
